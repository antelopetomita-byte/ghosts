01 — Concept

なにが違うのか

SignalやTelegramのような「消えるチャット」は全体を一括で自動削除する。Ghostsは 1通ごとに寿命を選べる ことを核にした。

メッセージの寿命
残す 既読後 › 10秒 1分 10分 1時間 24時間 7日
消滅するとサーバーからも実削除。相手の画面でも粒子になって消える。復元はできない。
02 — Journey

これまでの歩み

iPhone1台・コードはすべて委任で、ゼロから積み上げた開発の流れ。

Step 01 · 骨組み

画面遷移型アプリの土台

GitHub Pagesで公開。ENTER → 登録 → 一覧 → チャットの画面切替UIをスマホアプリライクに構築。

Step 02 · 世界観

ロゴ・アイコン・デザイン確立

粒子化するゴーストのロゴと起動画面、ホーム画面アイコン。黒×スペクトラルエメラルドの世界観に統一。

Step 03 · 招待制

招待した人だけが繋がる

ID検索による無断追加を廃止。招待リンク／QRを渡した相手が承認して初めて接続成立。

Step 04 · Firebase

本物のリアルタイム通信

匿名ログイン＋Firestoreで実際の送受信。消滅ルールでサーバー側も実削除する仕組みに。

Step 05 · アカウント

ニックネーム・ID・ログイン

表示名と自分で決めるID、パスワードを設定。別端末でもログインで同じIDに入れる（ID量産を解消）。

Step 06 · グループ

グループチャット

複数人の会話、送信者名表示、寿命設定にも対応。「既読後」は全員が読んだら消滅。

03 — Features

主要機能

匿名アカウント

電話番号もメールも不要。ニックネーム＋Ghost ID＋パスワードで、端末をまたいで同じIDに。

招待制

招待リンク／QRを渡した相手だけが接続可能。リンクは作り直して無効化もできる。

リアルタイムチャット

Firestoreで即時送受信。既読表示・吹き出しUI。送信者名はグループでも表示。

メッセージ寿命

残す／既読後／10秒〜7日。期限が来ると両者の画面で粒子になって消滅、サーバーからも削除。

グループチャット

連絡先から複数人を選んで作成。メンバー追加・退出、グループでも寿命設定が使える。

プライバシー設計

サーバー保存は最小限。消えた会話は復元不可。接続済みの相手しか会話に触れられない。

プロフィール写真

自分のアイコンを写真/画像に。端末側で圧縮して保存し、招待・連絡先経由で相手にも反映。

画像の送受信

写真を圧縮して送信、インライン表示＋タップで全画面。寿命・削除・既読も画像に対応。

既読と未読

DMは既読＋時刻、グループは「誰が読んだか」一覧。一覧には未読件数バッジを表示。

取り消し送信

自分のメッセージを長押し → 全員の画面から削除。入力中インジケータやリンク化も。

04 — How it works

仕組み

単一の index.html がFirebaseと直接やり取りし、GitHub Pagesから配信される。

App
Ghosts（HTML / CSS / JS）画面切替UI・寿命ロジック・粒子エフェクト。1ファイル完結。
Auth
Firebase Authentication匿名ログイン ＋ メール/パスワード（同じIDで複数端末ログイン）。
DB
Cloud Firestoreusers / contacts / invites / connections / groups / messages をリアルタイム同期。
Host
GitHub PagesContents API で push → 自動デプロイ。
05 — Connect

招待のながれ

招待された相手としか繋がらない。これが「招待制」の中身。

01招待を渡すリンク or QR
02承認相手がタップ
03接続成立双方向で繋がる
04会話開始ここで初めて送受信
06 — Security

守りかた

接続済みの相手だけ

Firestoreのセキュリティルールで、接続済みの2人／グループのメンバー以外は会話を読み書きできない。UIをすり抜けても無理。

消えたら復元不可

寿命が尽きたメッセージはドキュメントごと削除。ミラーにも本文を残さない。

E2EEは今後の課題

現状メッセージはルールで保護されているが平文保存。文言どおりの暗号化は次フェーズで実装予定。
